# 시연 시나리오 (Heungbuja)

## 개요
- 프로젝트명: 흥부자(Heungbuja) – 노인용 음악 체조 & 응급 신고 보조 서비스  
- 시연 시간: 약 5분  
- 시연 환경: 실제 기기(또는 마이크) + 사용자 화면(프론트) + 관리자 페이지(백오피스)

---

## 1. 음악 체조하기 기능 시연

### 1-1. 호출 및 인트로
**목표**: 음성 명령으로 음악 체조 흐름 전체를 보여준다.

**시연 스크립트(예시)**  
> "흥부야, 체조할래"

**화면/동작 흐름**
1. 사용자가 기기(또는 앱)에 대고 "흥부야 체조할래"라고 말한다.
2. STT를 통해 음성이 텍스트로 변환되고, 인텐트가 `음악 체조 시작`으로 인식된다.
3. 화면에 “체조를 시작할게요. 함께할 노래를 골라주세요.” 안내 문구와 함께 노래 목록이 나타난다.

### 1-2. 노래 선택
**목표**: 사용자가 체조에 사용할 곡을 선택하는 과정을 보여준다.

**화면/동작 흐름**
1. 추천 리스트(최근 사용/인기 곡 등)에서 2~3곡을 보여준다.
2. 사용자가 곡을 선택한다.  
   - 음성: “두 번째 노래 틀어줘”  
   - 또는 리모컨/터치로 선택
3. 선택된 곡 정보(제목, 가수)가 상단에 표시되고 “1절부터 체조를 시작할게요.” 라는 안내가 나온다.

### 1-3. 체조 진행 및 실시간 피드백
**목표**: 동작 인식과 실시간 피드백을 강조한다.

**화면/동작 흐름**
1. 음악이 재생되며, 화면에는 현재 동작(예: 팔 올리기, 옆구리 늘리기 등) 애니메이션/아이콘이 표시된다.
2. 카메라/센서가 사용자의 동작을 인식해, 매 박자마다 동작 정확도를 계산한다.
3. 동작 수행 시, 화면과 음성으로 실시간 피드백 제공
   - “잘하고 계세요!”  
   - “조금만 더 팔을 높이 올려볼까요?”  
   - “지금처럼 천천히 따라오세요.”
4. 화면에는 실시간 점수 바 또는 동작 정확도 색상(초록/노랑/빨강 등)으로 피드백이 시각화된다.

### 1-4. 1절 종료 및 점수 집계
**목표**: 1절 결과를 바탕으로 난이도 조절의 근거를 보여준다.

**화면/동작 흐름**
1. 1절이 끝나면 음악이 잠시 줄어들고, “1절 체조가 끝났어요.” 안내가 나온다.
2. 1절 평균 점수/완성도 요약을 보여준다.
   - 예: “팔 동작 82점, 몸통 동작 75점, 전체 평균 78점”
3. 화면 하단에 “다음 2절 체조 난이도를 조정하는 중입니다…” 메시지가 표시된다.

### 1-5. 2절 난이도 동적 조절
**목표**: 1절 점수 기반으로 2절 난이도가 자동으로 바뀌는 것을 보여준다.

**조절 로직(설명용)**  
- 평균 점수가 높은 경우(예: 80점 이상):  
  - 동작 속도/범위를 조금 더 크게, 난이도 ↑  
- 평균 점수가 낮은 경우(예: 60점 이하):  
  - 동작 개수/속도를 줄이고, 안내 멘트를 더 자주, 난이도 ↓  
- 중간인 경우:  
  - 현재 수준을 유지

**화면/동작 흐름**
1. 시스템이 “1절 점수를 바탕으로 2절 난이도를 조정했어요.”라고 안내한다.
2. 2절 체조가 시작되며,  
   - 난이도 ↑인 경우: “조금 더 힘차게 해볼까요?” 같은 멘트와 함께 동작이 살짝 빨라짐.  
   - 난이도 ↓인 경우: “천천히, 편안하게 따라오시면 돼요.” 멘트와 함께 동작 개수가 줄어듦.
3. 2절 종료 후, 전체 체조 결과 요약과 함께 “수고하셨어요!” 메시지를 보여주며 시연을 마무리한다.

## 2. 응급 신고 기능 시연

### 2-1. 호출 및 상황 설정
**목표**: 위급 상황에서 음성 한마디로 도움을 요청할 수 있음을 보여준다.

**상황 예시**  
- 사용자가 어지러움/통증을 느끼는 상황을 가정

**시연 스크립트(예시)**  
> "흥부야, 살려줘"

**동작 흐름**
1. 사용자가 “흥부야 살려줘”라고 말한다.
2. STT가 이를 인식하여, 응급 인텐트로 분류한다.
3. 즉시 응급 모드로 전환되며, 사용자 화면에 큰 글씨로  
   - “응급 상황으로 인식했어요. 잠시만 기다려주세요.”  
   - 현재 시간, 장소(설정된 시설명/호실) 등을 표시한다.

### 2-2. 실시간 응급 알림 서비스
**목표**: 관리자/보호자가 바로 알 수 있다는 점을 강조한다.

**동작 흐름**
1. 서버에서 응급 신고 엔티티(`emergency_reports`)가 생성되며, 상태는 `PENDING`으로 저장된다.
2. 동시에 관리자 웹/알림 시스템으로 실시간 푸시 또는 알림이 전송된다.
   - 예: “○○요양원 302호, 김흥부님 응급 호출 발생”
3. 사용자 화면에는 “관리자가 곧 확인할 거예요.”라는 멘트와 함께, 신고가 전달되었음을 시각적으로 보여준다.

### 2-3. 관리자 페이지에서 확인 및 정보 조회
**목표**: 관리자가 어떻게 상황을 파악하고 대응하는지를 보여준다.

**화면/동작 흐름 (관리자 페이지)**
1. 관리자 페이지 로그인 후, 상단 또는 대시보드에 **실시간 응급 알림 배지/리스트**가 표시된다.
2. 최신 응급 신고를 클릭하면 상세 화면으로 이동한다.
   - 사용자 정보: 이름, 생년월일, 담당 관리자, 연락처
   - 신고 내용: 트리거 문장(“흥부야 살려줘”), 전체 발화 내용(있을 경우)
   - 신고 시각, 시설/방 정보
3. 관리자는 상세 화면에서 조치 상태를 변경할 수 있다.
   - `CONFIRMED`: 실제 응급 상황으로 확인  
   - `RESOLVED`: 응급 처리 완료  
   - `FALSE_ALARM`: 오인 신고
4. 관리자 메모를 남길 수 있다.
   - 예: “낙상 의심, 119 호출 후 보호자 연락 완료”

**연계 설명(선택적)**  
- 해당 응급 신고 정보는 이후 통계/리포트 화면에서 조회 가능하며, 응급 발생 빈도나 시간대 분석에 활용된다.
